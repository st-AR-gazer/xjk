API Docs

        
Hosted base path: /Clip-To-Ghost


        

          
GET /health

          
Returns text/plain with ok.

        


        

          
POST /api/inspect

          
Submit multipart/form-data with a required map and optional candidate filters.

          
    
            
  • map: required .Map.Gbx file.
    • 
            
  • clipIndex, trackIndex, blockIndex: optional non-negative integer filters.
    • 
          

          
Returns JSON with a manifest-style payload listing discovered GPS clip candidates.

          
curl -X POST "https://tools.xjk.yt/Clip-To-Ghost/api/inspect" ^
  -F "map=@example.Map.Gbx"

        


        

          
POST /api/export

          
Exports ghosts using the selected template mode and optional candidate filters.

          
    
            
  • map: required .Map.Gbx file.
    • 
            
  • templateMode: shipped, custom, or blank.
    • 
            
  • templateGhost: required file when using templateMode=custom.
    • 
            
  • includeManifest: optional boolean.
    • 
            
  • clipIndex, trackIndex, blockIndex: optional non-negative integer filters.
    • 
          

          
Returns a single .Ghost.Gbx if one export is produced and manifest output is disabled; otherwise returns a zip archive.

          
This is the preferred mode when exporting a single GPS clip for ARL.

          
curl -X POST "https://tools.xjk.yt/Clip-To-Ghost/api/export" ^
  -F "map=@example.Map.Gbx" ^
  -F "clipIndex=0" ^
  -F "trackIndex=0" ^
  -F "blockIndex=1" ^
  -F "templateMode=shipped" ^
  -F "includeManifest=false" --output clip.Ghost.Gbx
