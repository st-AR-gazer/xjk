# xjk map admin frontend

The browser entrypoint only composes the map editor. Editor responsibilities live in `modules/`:

- `access.js` owns the session gate and safe login return URL.
- `state.js` owns editable snapshots, reset, and undo history.
- `geometry.js` contains pure grid, point, route, and scaling operations.
- `rendering.js` renders the lists, inspector, SVG map, and editor control state.
- `controls.js` owns selection, mutations, dragging, and DOM event binding.
- `export.js` creates, copies, and downloads read-only layout snapshots.

Production geometry remains generated by `/map-layout.js`; this page never writes the generated source. Keep pure state
and geometry behavior covered in `modules/editor.test.mjs`, and preserve `app.js` as a composition boundary.
